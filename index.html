    About

    CodobaZombie is a real time strategy game (RTS) or at least that is what we are trying to do :). It is a 3D game located in Cordoba, Argentina, and it is a open source project. We are currently developing this game by our owns, in our free times and giving the best of us and using of course open and free tools.

    We also think that this medium size game could be useful for other peoples to learn about which are the basic things you need when you make a game from zero (or at least, you can see the things that you probably want to avoid :)). I said a medium size game since of course this is not a AAA game because we have not the time, money, technology, people nor experience to do one of that kind, but probably you can think this like a CCC game :).

    Our objectives are:

    • Create a free-to-play opensource multi-platform (Windows and Linux at least) game using opensource and free tools.
    • Share all the information/knowledge about our experiences / problems about all the development process.
    • Try to do this development process as much professionally as we can.
    • Be a starting point for future developments (games).

    In a first instance, due to the time we have we will try to create a demo (one level with basic functionality) and then, if we get the support we need (sponsors / advertisers), get involved as a full time job and finish the full game :).

    Development

    We are making the game with:

    • C++ as main language.
    • OGRE as the render engine.
    • OpenAL as sound library.
    • ffmpeg for videos.
    • tinyxml for xml.
    • boost and stl.
    • cmake for building.
    • and a lot of other things.

    The idea is to make a well designed and commented code, follow and apply good programming practices, and of course, learn and share knowledge. We are currently in a transition stage trying to improve our "old" code and port it to use the new designed architecture and the new C++11 standard, unify the coding style and port the code to be more standard (we start the development in linux, so now we need to port it to be windows compatible).

    You can check in the other wikis for the Architecture / Coding Style / Development methodologies / etc. Some of this things are new and probably are not perfectly reflected in all our code, as just said before, we are in a transition stage and trying to improve all the Software development life-cycle :).
