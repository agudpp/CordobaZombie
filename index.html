    About

    CodobaZombie is a real time strategy game (RTS). It is a 3D game that takes place in Cordoba, Argentina, and it is a open source project. It is an open source project and we are currently developing this game with all of our best efforts in our free time.

    We believe that a game with a size of this caliber will be useful for others to learn how to make a game from absolutely nothing. We are not a professional team though many of us have applied experience. We aim to create a polished product.

    Our objectives are:

    • Create a free-to-play open-source multi-platform (Windows and Linux minimum) game using open-source tools.
    • Share knowledge and experience with others.
    • Manage a professional development process.
    • Become a starting point for future game developments.

    Our primary goal is to create a demo with a solid level and with core game elements. Once we receive the support we need (via sponsors and advertisers), we can become established as a full-time developer and finish the entirety game.

    Development

    We are making the game with:

    • C++ as main language.
    • OGRE as the render engine.
    • OpenAL as sound library.
    • ffmpeg for videos.
    • tinyxml for xml.
    • boost and stl.
    • cmake for building.
    • and a lot of other things.

    The design concept aims to make well-designed and documented code, follow and apply good programming practices; and of course; learn and share knowledge. We are currently in a transition stage trying to improve our "old" code and port it to use the new designed architecture and the new C++11 standard, unify the coding style and port the code to be more standard (we start the development in linux, so now we need to port it to be windows compatible).

    You can check in the other wikis for the Architecture / Coding Style / Development methodologies / etc. Some of this things are new and probably are not perfectly reflected in all our code, as just said before, we are in a transition stage and trying to improve all the Software development life-cycle :).
